  • Gilligan: an Infrastructure Automation Drone Plugin
    • CI/ CD Plugin
    • Drone/ Vela
    • Deployments-as-Code
    • Containerization
    • Ruby
    • Work
    • Architecture and Design
    • TODO: Image of a boat blueprint

    This project, while closed source, was by-and-large the highlight of my time consulting for Cargill. They were on their journey towards public-cloud adoption and were making great strides. An infrastructure team had been spun up to build Captain Cargill's development teams to define all of their deployments to AWS as code using a syntax fairly similar to Kubernetes manifest files. But there was one problem; due to networking constraints, not all workloads could go to AWS. For on-prem workloads, there was no deployment-as-code solution requiring engineers to learn a nuanced web interface for configuring VMs, and then navigate it everytime a new workload was needed. In a few weeks, with some of the amazing engineers at Cargill, I released Gilligan, a ruby-based CI/ CD plugin that served as a counterpart to the existing Captain solution.
